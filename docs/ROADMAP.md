# ロードマップ — 執筆支援エディタ（小説／ライトノベルモード）

> **技術スタック**: Kotlin Multiplatform（コア共通化）＋ Android / Web アダプタ＋ Ktor（バックエンド）
> **設計方針**: コア（ロジック）／アダプタ（UI・WASM・CLI）／MCP（後付け）の3層分離
> **運用方針**: MVP-first で素早く公開し、リリース後にプロダクトを深める。Web は PWA としてアプリ化し、ブラウザ利用とインストール利用の両対応を最低ラインとする

---

## 命名（再提示）

過去プロジェクト（`room-design`, `discord-vc-watcher-bot`, `claude-usage-bot`）の傾向：

- **ケバブケース**（小文字 + ハイフン区切り）
- **内容が読み取れる説明的な英語**（造語より「何をするか」優先）
- サービス名は内容を素直に表す（room-design → 「ルームデザイン」）
- 用途別サフィックス（`-bot` 等）で種別を明示

今回は単調さを避けるため、`novel` + `editor` を**かばん語（混成語）**にして一語へ溶かす。
構成は従来どおり **1プロダクト＝1リポジトリ**（モノレポ）。中を Gradle モジュールで分割する：

| 種別 | 名称 | 補足 |
|---|---|---|
| **リポジトリ／フォルダ名** | `noveditor` | ワンタン（一単語）で `cd` しやすい。novel + editor の混成。これ1つに全モジュールを内包 |
| **サービス名** | ノヴェディタ / Noveditor | 「小説 エディタ」近傍で検索に乗りつつ固有名詞感もある |
| **モジュール（共通コア）** | `core/` | KMP 共通モジュール（校正・整形ロジック）。プラットフォーム非依存 |
| **モジュール（Web）** | `web/` | Web / PWA アダプタ |
| **モジュール（Android）** | `android/` | Android アダプタ |
| **モジュール（バックエンド）** | `server/` | Ktor 製。同期・校正コアの API 化を担う |

> **なぜモノレポか**：KMP は Gradle マルチモジュールが前提で、コア変更とアダプタ変更を1コミット／1PR で通せ、`expect`/`actual` の対応がリポジトリをまたがず、CI も一本化できる。一人で1プロダクトを育てるフェーズでは分割の利点（コアの独立配布・権限分離・別リリースサイクル）が効かず、publish→バージョン上げ→取り込みの往復コストだけが残るため。コアを独立 OSS として切り出したくなった時点で分離すれば間に合う。
>
> 別案：`novedit`（より短い／動詞的）、`storiter`（story + writer で novel 非依存）など。SEO を強く狙うなら説明性を戻した `novel-writing-editor` 系も候補。

## リリース段階の全体像

| 段階 | コンセプト | 出すもの |
|---|---|---|
| **MVP** | まず書ける・保存できる | 最小エディタ + ローカル保存 |
| **v1** | 執筆支援が成立する | 校正コア + ライトノベルモード |
| **v1.x** | 育てる | フィードバック反映・整形強化 |
| **将来** | 拡張する | クラウド同期・MCP 対応・AI 連携 |

## MVP — 「まず書けて、消えない」

最小限で公開できる状態。執筆体験の土台だけを作る。

### 含むもの
- テキスト入力エリア（縦書き対応は将来、まずは横書き）
- ローカル保存（LocalStorage / Web、ファイル or DB / Android）
- 文字数・行数カウント
- 複数原稿の作成・切り替え（最小の一覧）
- 共通コア `core/` モジュールの初版（保存モデル・文字数ロジック）
- **PWA 対応（Web を最低限アプリ化）**：
  - Web App Manifest（名前・アイコン・スタンドアロン起動）
  - Service Worker（オフラインキャッシュ）
  - HTTPS 配信・インストール可能要件の充足
  - ローカル保存と組み合わせ、オフラインで「書く → 保存」が成立

### 含まないもの（割り切り）
- 校正・表記ゆれチェック → v1
- ライトノベル特有の表現支援 → v1
- クラウド同期 → 将来
- プッシュ通知・バックグラウンド同期（PWA の発展機能）→ 将来

### 完了条件（最初のリリース = Web/PWA）
- Web が PWA としてインストール可能・オフラインで起動できる
- Web で「書く → 保存 → 再度開く」が成立（Android ユーザーにはインストーラブル PWA で到達）
- コアロジックが `core/` モジュールに分離され、Web から再利用されている
- **ネイティブ Android は次リリース**（別 spec `specs/android/`・core 再利用。設計は本リリースと並行先行）

> **リリース順の補足**: 「素早く公開」を優先し、最初のリリースは Web/PWA に絞る。PWA は Android でインストール可能・オフライン動作するため、ネイティブアプリ無しでも Android ユーザーに届く。ネイティブ Android は KMP の共有 core を再利用してアダプタ層（Compose UI ＋ 端末保存）だけを足す形で次リリースに回す。

## v1 — 「執筆支援が成立する」

ただのエディタから「執筆“支援”エディタ」へ。ここがプロダクトの核。

### 含むもの
- **校正コア**：表記ゆれ検出（全角/半角、三点リーダー「…」の数、「！？」連続、句読点の統一）
- リアルタイム指摘（入力に応じて警告を Flow で流す）
- **ライトノベルモード**：
  - ルビ記法のサポート（`|漢字《かんじ》` 等）
  - 会話文・地の文の視認補助
  - セリフ比率・文長の可視化
- 原稿のエクスポート（プレーンテキスト / Markdown）
- フィードバック導線（意見投稿リンクを UI に常設）

### 完了条件
- 校正コアが共通モジュールとして両プラットフォームで動作
- ライトノベルモードの ON/OFF で支援内容が切り替わる
- ユーザーがフィードバックを送れる導線がある

## v1.x — 「育てる」

リリース後のフィードバックを反映する継続フェーズ。

### 候補（優先度は反応次第）
- 校正ルールの追加・カスタマイズ（ユーザー定義の表記辞書）
- 整形機能の強化（インデント、字下げ自動化、ダッシュ「──」整形）
- 縦書き表示モード
- テーマ／フォント設定（長時間執筆の快適性）
- 自動保存・履歴（簡易バージョン管理）

### 進め方
- フィードバックをカテゴリ分けし、影響度の高いものから着手
- ROADMAP.md のこの節を都度更新

---

## 将来 — 「拡張する」

プロダクトを広げる長期構想。設計上「後付けできる形」だけ MVP/v1 で確保しておく。

- **クラウド同期**：端末間で原稿を同期（`server/` モジュール（Ktor）を導入）
- **MCP 対応**：校正コアを MCP サーバー化し、外部 AI ツールから呼べるように（Ktor 上に MCP 層を実装。3層分離の MCP 層をここで具体化）
- **AI 連携**：推敲提案・続き生成の補助（コアとは疎結合に）
- **iOS 展開**：KMP の利点を活かしたプラットフォーム追加
- **共同編集**：複数人での原稿レビュー

## 既存エディタ調査の反映（機能バックログ）

> 海外（iA Writer / Ulysses / Scrivener / Typora 等）・日本語小説ツール（Nola / TATEditor / 縦式 / Hakogaki / でんでん 等）・執筆UXパターンを横断調査し、「落ち着いた原稿用紙・オフライン・私的」という方針で取捨選択した結果。

### 最初のリリースに追加（実装済み — クイックウィン＋シグネチャ）
- 原稿用紙換算（枚数 = ⌈字数/400⌉ を併記）
- スクロール/カーソル位置の記憶（原稿ごと）
- 集中モード（⌘\ で chrome 非表示）
- テーマ light / dark / **sepia（＝原稿用紙）**
- 文字サイズ・行幅の調整
- セッション字数（＋N字）
- **タイプライタースクロール**（本文の現在行を中央維持）
- **ルビ記法プレビュー**（`｜漢字《かんじ》` → `<ruby>`。read-only プレビュー）

### v1（執筆支援＝校正＋ラノベモード）
- 約物整形 lint（`……`/`――` の2個1組、全角/半角、`！？`連続、句読点統一、禁則）
- 傍点 `《《…》》` → `text-emphasis`
- Find & Replace（⌘F・原稿横断）
- **小説→話（連載）構造**：一話完結でない小説を 小説(Novel)→話(Episode) の階層で管理（カクヨム／なろう参考。詳細 [specs/novel-episodes/](../specs/novel-episodes/design.md)）
- フォーカス減光（段落以外を淡色化・任意ON）
- 原稿ごとの字数目標／読了時間目安
- 投稿サイト向けエクスポート（なろう/カクヨム記法・txt）

### v1.x（育てる）
- **縦書き・書籍化（PDF/印刷）＝ [tatemd](https://github.com/torifonium/tatemd) 連携で提供**（縦書き組版・禁則・A5/B6 書籍 PDF・絵巻長尺は tatemd を npm ライブラリ化して組み込み。**縦書き/組版は自前実装しない**。tatemd 側の embeddable 化 issue を起票して進める）
- 版履歴スナップショット（IndexedDB・差分・復元）
- キャラクター設定・あらすじ/プロットメモ（Nola 型）
- 穏やかな日次記録（「今週◯日書いた」程度・罰則なし）
- ⌘K コマンドパレット / 原稿クイック切替
- 文章分析（セリフ比率・文長・漢字率）
- EPUB 等の書籍化出力（tatemd 連携の発展）

### 外部連携方針
- **縦書き・組版・書籍化 PDF は [tatemd](https://github.com/torifonium/tatemd)（自作 OSS・MIT）に委譲**する。tatemd は 3 ティア（Web/拡張/CLI）が同一 `core` を共有する設計で、npm ライブラリ化すれば noveditor から縦書きレンダラ＋書籍 PDF を二重実装なしに利用できる。tatemd 側の「他サービスから組み込めるようにする」issue（npm 公開・browser ESM エントリ・縦書き CSS 配布・Web Component・埋め込み例）を起票して連携を実現する。

### あえて入れない（calm を守るための取捨）
- タイプ音 → 入れても将来の設定の奥に opt-in 止まり（既定OFF）
- 罰則つきストリーク（不安を煽る）→ 穏やかな日次記録で代替
- SNS共有・ゲーミフィケーション（private・offline-first と矛盾）
- 本格コルクボード/アウトライナ（“管理ツール”化＝スコープ膨張）
- ライブ縦書き編集（IME 崩れ）→ 縦書きはプレビューで代替

## 設計上の不変方針（全段階共通）

- **spec はアダプタごとに分割し、core を共有**：`specs/core/`（共有ロジック）／`specs/web/`（Web/PWA）／`specs/android/`（ネイティブ）。各アダプタ spec は core spec を参照し、ツールチェーン・リリースサイクルの密結合を避ける
- コアはプラットフォーム非依存の純粋ロジックに保つ（UI・I/O はアダプタ側）
- 各リポジトリに ROADMAP.md を配置し、段階の進捗を追記
- フィードバック導線は MVP から組み込み、リリース後に育てる前提で運用
- MCP 対応は「いつでも後付けできる」疎結合設計を最初から維持
