# クイックアクション・ヘルプ・検索（⌘K）— 設計

> 作成日: 2026-06-27
> 対象: noveditor/web（TS + Vite + Svelte 5）。core 変更は最小（必要なら commonMain に純関数を足す方向）。
> 方針: calm / offline-first を維持。UI を見ながら反復調整する前提の漸進実装。

## 目的

PC で快適に書くための導線を整える。具体的には次の5点を追加・改善する。

1. **コマンドパレット（⌘K）** — 操作＋（タイトル＋本文全文）検索の単一入口。
2. **ヘルプ（`?`）** — ショートカット一覧＋記法＋簡単な使い方。
3. **初回オンボーディング** — 一度きりのウェルカム＋空状態ガイド。
4. **「紙」テーマの紙質** — ただの白でなく、iPad ペーパーライク的な地合いを付与。
5. **PC 向けショートカット拡充** — ⌘K / `?` 以外も含めた執筆向けバインド。

## 既存の前提（調査済み）

- グローバルショートカットは `web/src/App.svelte` の `window` keydown ハブに集約（現状 ⌘/Ctrl 必須ゲート）。既存: ⌘S 保存 / ⌘N 新規話 / ⌘\ 集中モード。
- `AppState`（`web/src/state/appState.svelte.ts`, Svelte 5 runes）はメモリに **小説サマリ全件**＋**現在小説の話サマリ**を保持。**本文は現在話だけ**メモリにある → 本文全文検索は repo から本文を走査する必要がある。
- ルビプレビュー（`web/src/editor/renderRuby.ts`）・圏点 `《《…》》` は **web/TS 側**にある（core ではない）。
- 削除は確認ダイアログ無し＋「元に戻す」トースト（非ブロッキング）。modal は原則回避。
- テーマ token は `web/src/app.css`（`:root`＝紙 / `[data-theme='sepia']` / `[data-theme='dark']`）。紙は `--paper:#f7f4ee` だが surface との差が僅かで白く見える。

## 共通方針

- パレット/ヘルプは **軽量オーバーレイ**（Esc / 外側クリック / 同キー再押下で閉じる）。削除確認のようなブロッキング modal ではない＝calm 方針と矛盾しない。
- フォーカストラップ＋`aria`。IME はパレット入力欄で正常（新規 input なので崩れ無し）。
- 操作の実体は **`web/src/ui/commands.ts`** に一本化。パレット・ショートカット・ヘルプが**同じコマンド定義を共有**＝表記ずれ防止。

## ① コマンドパレット（⌘K / Ctrl+K）

- 新規 `web/src/ui/CommandPalette.svelte`。単一入力欄＋結果リスト。
- 2種を1欄で扱う:
  - **操作（コマンド）**: 新規話・新規小説・テーマ切替（紙/セピア/夜）・集中モード(⌘\)・設定を開く・タイプライタースクロール切替・ヘルプを開く・現在の話を削除。
  - **移動/検索**: 小説タイトル・話タイトル・**本文全文**にマッチ → Enter でその話を開く（スニペット表示）。
- `commands.ts`: `{id, label, hint?, keywords?, run()}` 配列。`AppState`/`settings` を受け取り `run()` で実行。純データ＋関数でテスト可能。
- 検索実装:
  - タイトル: メモリ上サマリ＋全小説を `repo.listEpisodes` で走査（軽量）。
  - 本文: `NovelRepository` に `searchEpisodes(query): Promise<{episodeId, novelId, snippet}[]>` を追加し本文走査（現実的件数なので素直なスキャンで十分。将来 index 化余地）。
- 並び: 完全一致 ＞ 前方一致 ＞ 部分一致。コマンド/検索はセクション分け。↑↓ 移動・Enter 実行・Esc 閉じる。

## ② ヘルプ（`?`）

- 新規 `web/src/ui/HelpOverlay.svelte`。
- 起動: **`?` キー（修飾なし・入力欄にフォーカスが無い時だけ）** / パレットの「ヘルプ」 / ヘッダの小さな `?` ボタン。
- 中身: **ショートカット一覧**＋**記法**（ルビ `｜漢字《かんじ》`・圏点 `《《…》》`）＋**簡単な使い方**（＋で話を追加 → 書くと自動保存 → 削除はトーストでUndo → ⌘K で検索）。
- 閉じる: Esc / `?` 再押下 / 外側クリック / ×。

## ③ 初回オンボーディング

- フラグ `noveditor:onboarded:v1`（localStorage）で初回のみ判定。
- **(a) 空状態ガイド**（原稿が空なら常時）: 本文エリア中央に「最初の小説を作る ＋」CTA＋「⌘K で検索・`?` でヘルプ」の小さな案内。
- **(b) 初回ウェルカム**（一度きり）: 穏やかな小カードで 3 ステップ（＋で話を追加 / 書くと自動保存 / ⌘K・`?`）。閉じるとフラグ保存で再表示しない。overlay でなくエディタ内に溶かす。

## ④ 「紙」テーマの紙質

- **オフライン厳守**（外部画像なし）。**インライン SVG `feTurbulence` ノイズを data-URI 化**し、`body::before` 固定オーバーレイに **極低 opacity（〜3–5%）** で敷く＝紙の地合い（tooth/grain）。
- 併せて: 紙トーンを僅かに暖色寄り・surface との階調差を少し付与・ページ端のごく淡い陰（vignette）で「紙の面」を感じさせる。
- テキスト可読性最優先（ノイズは本文の上に乗せない）。セピアにも軽く適用、夜は無し/別調整。
- **Chrome / Safari / Firefox で確認**。Chrome 専用プロパティ不使用。

## ⑤ PC 向けショートカット拡充

現状: ⌘S 保存 / ⌘N 新規話 / ⌘\ 集中モード。追加:

- **⌘K** コマンドパレット / **`?`** ヘルプ
- **⌘⇧N** 新規小説
- **⌘B** サイドバー開閉
- **⌘⌥↑ / ⌘⌥↓** 前の話 / 次の話（執筆中の移動）
- いずれも `preventDefault` でブラウザ既定と衝突回避。`App.svelte` keydown ハブに集約（`?` のみ入力フォーカス判定の別経路）。バインド詳細は UI で確認しつつ調整（衝突するものは差し替え可）。

## テスト

- `commands.ts`（純データ/関数）と `searchEpisodes`（repo）を中心に vitest。
- テーマ質感は視覚確認（3ブラウザ）。
- 既存テスト（core jvmTest + web vitest 計23件）を緑のまま維持。`pnpm build` 緑を維持。

## 実装順（UI を見ながら調整）

1. ④ 紙テーマ質感（最速で視覚確認・単独で安全）
2. ⑤ ショートカット拡充 ＋ ②ヘルプ（ショートカット一覧の出力先）
3. ① コマンドパレット（`commands.ts` を ②⑤ と共有）
4. ③ オンボーディング（空状態＋初回ウェルカム）

各ステップごとにブラウザ確認 → 調整 → 次へ。
