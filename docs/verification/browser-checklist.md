# ブラウザ最終確認チェックリスト — noveditor 最初のリリース（Web/PWA）

> 目的: ROADMAP「最初のリリース = Web/PWA」完了条件と各 US の受け入れ基準を、実ブラウザで通しで確認する。
> 手順: `./gradlew :core:jsBrowserProductionLibraryDistribution`（core ビルド）→ `web/` で `pnpm install && pnpm build && pnpm preview`（または `pnpm dev`）→ 下記を上から確認。
> 記号: ☐ 未確認 / ☑ OK / ✗ 要修正（メモ欄に状況）

## A. 書く・保存・開き直す（US-001 / 003 / 004）
- ☐ エディタに本文を入力すると即時反映される
- ☐ 編集を止めると**自動保存**される（800ms 後）／**明示保存**ボタンでも即保存
- ☐ ページを再読み込みすると、最後に編集した原稿が復元される
- ☐ 起動時、保存済みが複数あれば **updatedAt 最大（最新）の原稿**が開く
- ☐ 保存データが無い初回は空の初期状態

## B. 文字数・行数（US-002）— ⚠️ JS実機での正確性確認（commonTest は JVM のみ実施）
- ☐ 半角 `hello` → 5 文字
- ☐ 全角 `こんにちは` → 5 文字
- ☐ **全角スペース** `あ　い`（U+3000）→ 総 3・空白除外 2（**U+3000 が空白として除外されるか＝JS の `isWhitespace` 挙動確認**）
- ☐ **絵文字** 😺（サロゲートペア）→ 1 文字（2 ではない）
- ☐ 改行のみ `\n\n\n` → 行数 3・段落 0
- ☐ CRLF 混在でも行数が正しい
- ☐ 段落（空行区切り）が想定どおり数えられる

## C. 日本語 IME（US-001）
- ☐ かな漢字変換**中**（未確定）は文字数が増えない
- ☐ 変換**確定**後に文字数が正しく加算される

## D. 複数原稿・タイトル・削除（US-005 / 007 / 008）
- ☐ 新規作成で空の原稿が増える（一意 id）
- ☐ 原稿を切り替えると、切替前の編集が保存され、選択原稿が復元される
- ☐ タイトルを編集すると一覧表示に反映される
- ☐ 削除（確認あり）で原稿が一覧・ストレージから消える
- ☐ 編集中の原稿を削除したら、別原稿（無ければ空初期状態）へ遷移

## E. PWA / オフライン（US-006）
- ☐ DevTools → Application で **manifest** が読める（name/icons/standalone）
- ☐ DevTools → Application で **Service Worker** が registered
- ☐ DevTools **Offline** にしてリロード → アプリシェルから起動できる
- ☐ オフライン状態で「書く → 保存」が成立する
- ☐ **インストール可能**（アドレスバーのインストール導線 or Lighthouse PWA: installable）

## F. 異常系（US-003）
- ☐ ストレージ保存失敗（DevTools で localStorage 容量超過を模擬 等）時、**失敗が通知**され、編集中の内容は破棄されない
- ☐ index と本文の不整合を手で作って（DevTools で片方削除）リロード → 自己修復され起動継続
